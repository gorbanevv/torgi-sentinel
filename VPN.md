# Вариант без телефона: torgi через свой VPN (Sota) на VPS

Идея: бот живёт на VPS, но запросы к torgi.gov.ru уходят через твой Sota VPN
(его IP — не Timeweb). Telegram и всё прочее — напрямую. **Работает только если
выходной IP Sota не в бане у torgi** — это проверяется скриптом за минуту.

## Безопасность ключа
Твой ключ Sota **я не вижу**. Ты кладёшь его в файл на VPS сам, из своего ssh.
Скрипты (`scripts/vpn/*`) только читают этот файл в конфиг xray и **никогда не
печатают** ни ключ, ни хост — можешь прочитать их и убедиться. Конфиг xray и файл
ключа лежат с правами 600 в `/etc/torgi-sentinel/` и в git не попадают.

## Шаг 1. Положи ключ Sota в файл (в СВОЁМ ssh — я это не вижу)
Возьми ключ Sota (в Telegram-боте Sota → профиль/ключ, или на sotavpn.com).
Если есть выбор страны — выбери **Россию**. Затем на VPS:
```bash
umask 077
mkdir -p /etc/torgi-sentinel
cat > /etc/torgi-sentinel/sota.key      # вставь ключ, затем Enter и Ctrl+D
chmod 600 /etc/torgi-sentinel/sota.key
```
Ключ может быть вида `vless://…`, `ss://…`, `trojan://…`, `vmess://…` или
ссылка-подписка `https://…` — скрипт разберётся сам.

## Шаг 2. Скажи мне «положил» — я запущу проверку
Я выполню на VPS:
```bash
cd /opt/torgi-sentinel && git pull
bash scripts/vpn/proxy-up.sh
```
Скрипт поставит xray, поднимет локальный SOCKS через Sota и проверит torgi.
Вывод — либо «✅ РАБОТАЕТ», либо «❌ не работает» (тогда IP Sota тоже в бане —
попробуй сменить локацию Sota и обнови ключ в файле).

## Шаг 3 (если РАБОТАЕТ). Подключаю бота к прокси
Бот начнёт ходить на torgi через `socks5://127.0.0.1:10808`, Telegram — напрямую,
и будет крутиться на VPS как systemd-сервис 24/7. Телефон не нужен.
